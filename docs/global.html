    DropdownSearch(props)

    This component will act as a select dropdown input field, where the user can click on it to expand a menu listing all of the different options, and when clicking on any of these options it will change the state of the component to the selected option. This component has the ability to search and filter for a specific string to find an option more easily. This component will also have the feature to create a new option when using the search field and not finding an exact match.

    Parameters

    • props Object

      Component props object

      Properties

      • defaultText String <optional>

        This is the text the component will display when there is no option selected. If none is provided the default text will be "Click to select an option"

      • searchPlaceholder String <optional>

        This is the text the component will display in the input field for searching options. If none is provided the default text will be "Search for an option"

      • optionsList Array.<{id:Number, value:string}>

        The list of options the component will provide.

      • initialSelection null | Number <optional>
         null

        If provided a number, the component will have the option selected which id is equal to this value. Otherwise none will be selected.

      • onSelectCallback function null|Number

        A callback function that will be triggered when the user selects an option, it will receive the id of the option selected (or null if none is).

      • onCreateCallback function String

        A callback function that will be triggered when the user creates an option, it will receive the name of the option and return the id for it. It was designed this way based on the specific logif of this project.

      • resetTrigger Number <optional>
         0

        This is a prop that will be used to trigger a useEffect hook from an outside component so that the option can be set back to null and effectively reset the selection. It has a default value of 0, so that the first render doesn't execute the reset value (and overwrite the initialSelection).

    EditableDuration(props)

    This component will display a time, not as in a time in a 24h day but as in an arbitrary amount of time in the same shape of Hours, Minutes and Seconds, used for displaying the total duration of any task calculated using the start and end timestamps. When clicked, this component will allow the user to edit the value with another component that will act as an input for this custom format, so that it constraints the input to a valid format.

    InputDuration(props)

    This component will create an input element of type text, which we will control the position of the caret (cursor) and the values that are inputed, so that it always represents a valid time in the form of H:MM:SS, including the colon signs. To keep track of the position of the caret we will use the input properties of selectionStart and selectionEnd, and we will use the event onKeyDown to receive the numeric inputs from the user and handle them appropriately, and the logic that comes with the constrainst of the format, for example, that seconds and minutes are 2 digits numbers and cannot be larger than 59, etc.
