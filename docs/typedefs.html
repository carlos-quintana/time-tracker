    Type Definitions

    CurrentTask

    This is one particular task that will be active. This means it will not have a finish timestamp and it will be displayed in the timer. This task is assigned anytime the timer is started, and storing this data will effectively allow the user to start a task, close the applcation and have it persist the next time it's opened again.

    Type

    • Object

    Properties

    • name String

      The text description of the task.

    • start Number

      The UNIX timestamp that represents the moment the task begins.

    • project Number <optional>

      The numeric id of the project this task belongs to.

    Task

    This is the fundamental element that this application works with. A task is something the user has worked on for any amount of time.

    Type

    • Object

    Properties

    • id Number

      The unique identifier of the task in the database for the user.

    • name String

      The text description of the task.

    • interval Interval

      An object containing the start and finish UNIX timestamps for the duration of the task.

    • project Number <optional>

      The numeric id of the project this task belongs to.
